# Sleep and Wake

*Stub: written after v0.2.0.*

The finale: the moment Kaalm stops looking like a fancy way to run a
container.

- Watching the idle agent hibernate: the Pod disappears, the agent resource
  stays, storage stays.
- Sending a message to a sleeping agent and watching it wake, answer, and
  remember the earlier conversation.
- What this means for running many agents cheaply: compute only while
  someone is talking.
